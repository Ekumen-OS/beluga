Beluga named requirements: MotionModel

Requirements for a motion model to be used in a Beluga ParticleFilter.

Requirements

A type T satisfies the MotionModel requirements if the following is satisfied:

  • T::state_type is a valid type, representing a particle state.
  • T::update_type is a valid type, representing a motion model update.

Given:

  • An instance p of T.
  • A possibly const instance cp of T.
  • A possibly const instance of T::update_type u.
  • A possibly const instance of T::state_type s.

Then:

  • p.update_motion(u) will update the motion model with u. This will not actually update the particle states, but the update done here will be used in subsequent calls to the apply_motion() method.
  • cp.apply_motion(s) returns a T::state_type, that is the result of applying the motion model to s based on the updates.
  • cp.latest_motion_update() returns a std::optional<update_type> with the latest motion update received through motion_update().
