Detailed Description

template<class Mixin>
class beluga::BootstrapParticleFilter< Mixin >

Base implementation of a particle filter.

This class implements BaseParticleFilterInterface and satisfies the Beluga named requirements: BaseParticleFilter.

Template Parameters
MixinThe mixed-in type. An instance m of Mixin must provide:
  • A states(), weights() and initialize_particles() methods that satisfy the requirements specified in Beluga named requirements: StoragePolicy.
  • A generate_samples() method that satisfies the requirements specified in Beluga named requirements: StateGenerator.
  • A generate_samples_from_particles() method that satisfies the requirements specified in Beluga named requirements: Sampler.
  • A take_samples() method that satisfies the requirements specified in Beluga named requirements: Limiter.
  • An apply_motion() method that satisfies the requirements specified in Beluga named requirements: MotionModel.
  • An importance_weight() method that satisfies the requirements specified in Beluga named requirements: SensorModel.

void beluga::BootstrapParticleFilter< Mixin >::resample ( )
inlinefinal

Resample particles based on their weights.

This step creates a new particle set drawing samples from the current particle set. The probability of drawing each particle is given by its importance weight.

The resampling process can be inhibited by the resampling policies.

If resampling is performed, the importance weights of all particles are reset to 1.0.

The update will be done based on the Mixin implementation of the Sampler and Limiter named requirements and the active resampling policies.

Update the weights of the particles.

This step computes the importance factor or weight of each particle to incorporate measurements. The importance is proportional to the probability of seeing the measurement given the current particle state.

The update will be done based on the Mixin implementation of the SensorModel named requirements.
